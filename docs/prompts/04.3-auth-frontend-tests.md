# 04.3 — Authentication: Unit Tests (Frontend, TDD Red Phase)

You are a Senior Angular Engineer following Test-Driven Development (TDD).

The project follows a Feature-Based Architecture and the Angular Style Guide.

## Your task

Write **only** the unit tests for the `Login` component and `AuthService`.

Do **not** generate any production component code.

The tests must **fail** because the production code does not exist yet (red phase).

## Files the tests will target (stubs only)

Place the feature at:

```
frontend/src/app/src/features/auth/login/
  login.ts              ← standalone component
  login.html
  login.css
  login.spec.ts         ← tests only in this prompt

frontend/src/app/src/core/services/
  auth.service.ts       ← injects HttpClient, handles login + token storage

frontend/src/app/src/core/guards/
  auth.guard.ts         ← protects routes that require authentication
```

## AuthService contract

```ts
class AuthService {
  login(email: string, password: string): Observable<{ token: string }>
  saveToken(token: string): void       // saves to localStorage
  getToken(): string | null
  isAuthenticated(): boolean           // true if token exists and is not expired
  logout(): void                       // removes token from localStorage
}
```

## Business requirements

- `email` is required and must be a valid email format.
- `password` is required.
- The **Login** button must be disabled while the form is invalid.
- While the request is in flight, a loading indicator must be shown.
- On success: save the token via `AuthService.saveToken()` and navigate to `/tasks`.
- On `401 Unauthorized`: display `"Invalid email or password."`.
- On any other error: display `"An unexpected error occurred. Please try again."`.
