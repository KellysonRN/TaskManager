## Validation rules in `LoginHandler`

- `Email` null or whitespace → throw `ValidationException("Email is required")`.
- `Password` null or whitespace → throw `ValidationException("Password is required")`.
- User not found → throw `UnauthorizedException("Invalid credentials")`.
- Password does not match → throw `UnauthorizedException("Invalid credentials")`.

> Use the same generic message for "not found" and "wrong password" to avoid user enumeration (OWASP A07).
