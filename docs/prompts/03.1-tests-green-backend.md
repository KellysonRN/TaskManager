## Validation rules to implement in `CreateTaskHandler`

- Title is required (null or whitespace → throw).
- Title max 200 characters.
- Description max 1000 characters.
- DueDate cannot be in the past.
- Status must be one of: `Pending`, `InProgress`, `Completed`.

## Constraints

- Use **FluentValidation** (`AbstractValidator<CreateTaskCommand>`) for validation — do **not** use manual if/throw blocks in the handler.
- Domain entity properties must be **non-nullable** (`string`, not `string?`) — enforce required fields.
- `CreateTaskHandler` must only depend on `ITaskRepository` and `IUnitOfWork` — no concrete types.
