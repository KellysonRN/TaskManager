# 03.2 — Create Task: Refactor + Expose Endpoint (Backend)

All tests are green.

## Your task

1. Refactor the implementation without changing behaviour.
2. Create the `POST /api/tasks` REST endpoint.
3. Also create the `GET /api/tasks` endpoint (list all tasks).

Do **not** break existing tests.

## Refactor checklist

Look for and fix:

- Duplicate code
- Long methods
- Poor naming (unclear variables, types, methods)
- SOLID violations (especially SRP and DIP)
- Unnecessary abstractions
- Clean Architecture violations (e.g., infrastructure types leaking into Application)

## Endpoint requirements

### `POST /api/tasks`

- Controller: `TasksController` in `TaskManager.Api/Controllers/`
- Route: `POST /api/tasks`
- Request body: `CreateTaskCommand`
- On success: return `201 Created` with the `TaskDto` in the body
- On `ValidationException`: return `400 Bad Request` with error details

### `GET /api/tasks`

- Route: `GET /api/tasks`
- Response: `200 OK` with `IReadOnlyList<TaskDto>`
- Requires `GetAllTasksQuery`, `GetAllTasksHandler`, `GetAllTasksQuery` in Application layer
- Requires `TaskRepository.GetAllAsync()` implementation in Infrastructure

## Architecture constraints

- `TasksController` must depend on **interfaces** (`IRequestHandler<,>` or dedicated handler interfaces), not concrete handler classes.
- Register all handlers and repositories in the `DependencyInjection.cs` of their respective layers.
- `ExceptionHandlingMiddleware` must translate `ValidationException` to `400 Bad Request`.

## For each refactoring, explain

- What was wrong.
- What you changed.
- Why the change improves the code.
