# 04.4 — Authentication: Frontend Component + Full Integration (Green Phase)

All frontend unit tests from `04.3` are currently failing. Implement the production code.

Do **not** break any existing tests.

## Part A — Login Component

Create the standalone Angular component:

```
frontend/src/app/src/features/auth/login/
  login.ts
  login.html
  login.css
```

### Component requirements

- Standalone component using `ReactiveFormsModule` and Angular Material.
- Fields: `email` (type="email"), `password` (type="password").
- Login button disabled while form is invalid.
- Show loading indicator (`mat-progress-spinner` or `[data-testid="loading"]`) while request is in flight.
- On success: call `AuthService.saveToken(token)` then `Router.navigate(['/tasks'])`.
- On `401`: show `"Invalid email or password."`.
- On any other error: show `"An unexpected error occurred. Please try again."`.
- Responsive layout with Angular Material (`mat-form-field`, `mat-input`, `mat-button`).

## Part B — AuthService

Create `frontend/src/app/src/core/services/auth.service.ts`:

```ts
@Injectable({ providedIn: 'root' })
export class AuthService {
  private readonly tokenKey = 'auth_token';

  login(email: string, password: string): Observable<{ token: string }>
  saveToken(token: string): void       // localStorage.setItem
  getToken(): string | null            // localStorage.getItem
  isAuthenticated(): boolean           // token exists (basic check — no JWT decode required)
  logout(): void                       // localStorage.removeItem + navigate to /login
}
```

The `apiUrl` must be read from Angular environment files (`environment.ts` / `environment.prod.ts`).

---

## Part C — Auth Guard

Create `frontend/src/app/src/core/guards/auth.guard.ts`:

- Implement as a functional guard (`CanActivateFn`).
- If `AuthService.isAuthenticated()` returns `true` → allow navigation.
- Otherwise → redirect to `/login`.

---

## Part D — HTTP Interceptor

Create `frontend/src/app/src/core/interceptors/auth.interceptor.ts`:

- Implement as an `HttpInterceptorFn`.
- Attach `Authorization: Bearer <token>` header to every outgoing request when a token exists.
- Register the interceptor in `app.config.ts` (or `app.module.ts`).
