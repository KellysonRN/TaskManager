All frontend unit tests are green and backend endpoints exist.

## Part A — Backend Integration Tests

Write integration tests that exercise the full HTTP stack using `WebApplicationFactory<Program>`.

### Setup requirements

- Use `WebApplicationFactory<Program>` from `Microsoft.AspNetCore.Mvc.Testing`.
- Override the EF Core database to use an **in-memory SQLite** database (not the production database).
- Reset the database state before each test to avoid inter-test pollution.
- Place tests in `backend/tests/TaskManager.IntegrationTests/`.

### Test scenarios to cover

| # | Scenario | Expected HTTP Status |
|---|---|---|
| 1 | `POST /api/tasks` with valid payload | `201 Created` + `TaskDto` in body |
| 2 | `POST /api/tasks` with empty title | `400 Bad Request` |
| 3 | `POST /api/tasks` with past due date | `400 Bad Request` |
| 4 | `GET /api/tasks` returns list of tasks | `200 OK` + array |
| 5 | `GET /api/health` returns healthy | `200 OK` |

### Assertions

Use **Shouldly** for all assertions.

## Part B — Frontend Component (Make Tests Pass)

All Angular unit tests from `03.3` are currently failing. Implement the production component.

### Files to create

```
frontend/src/app/src/features/tasks/create-task/
  create-task.ts      ← standalone Angular component
  create-task.html    ← Angular Material form
  create-task.css     ← responsive styles

```

### Component requirements

- Use Angular **standalone component** with `ReactiveFormsModule` and Angular Material.
- `TaskService` must use `HttpClient` with the configured backend URL.
- After a successful save navigate to `/tasks` using `Router.navigate`.
- Show a loading spinner (`mat-progress-spinner`) while the HTTP call is in flight.
- Show a success message on `201 Created`.
- Show the API error message on `400`/`422`; show a generic message on `5xx`.

### Routing

Register the component in `app.routes.ts`:

```ts
{ path: 'tasks/create', component: CreateTask },
{ path: 'tasks',        component: ListTasks  },
```
