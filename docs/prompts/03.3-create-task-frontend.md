# 03.3 — Create Task: Unit Tests (Frontend, TDD Red Phase)

You are a Senior Angular Engineer following Test-Driven Development (TDD).

The project follows a Feature-Based Architecture and the Angular Style Guide.

The frontend already contains the project structure at `frontend/src/app/src/features/tasks/`.

## Your task

Write **only** the unit tests for the `CreateTask` component.

Do **not** generate any production component code.

The tests must **fail** because the production component does not exist yet (red phase).

## Component contract the tests will verify

Place the component at:

```
frontend/src/app/src/features/tasks/create-task/
  create-task.ts          ← standalone component
  create-task.html
  create-task.css
  create-task.spec.ts     ← tests only in this prompt
  task.service.ts         ← injectable service with create() and getAll()
```

## Business requirements

- `title` is required; max 200 characters.
- `description` max 1000 characters (optional field).
- `dueDate` cannot be in the past.
- `status` must be one of: `Pending`, `InProgress`, `Completed`.
- The **Save** button must be disabled while the form is invalid.
- Clicking Save with a valid form must call `TaskService.create()` exactly once.
- While saving, a loading indicator must be shown.
- After a successful save, navigate to `/tasks` (task list).
- If the API returns an error, display a user-friendly error message.
- The form must use Angular Material components (`mat-form-field`, `mat-select`, `mat-button`, `mat-progress-spinner`).
- The UI must be responsive.

## Test scenarios (cover all of these)

| # | Scenario |
|---|---|
| 1 | Component is created successfully |
| 2 | Form initializes with all fields empty |
| 3 | `title` marked invalid when empty |
| 4 | `title` marked invalid when exceeds 200 characters |
| 5 | `description` marked invalid when exceeds 1000 characters |
| 6 | `dueDate` marked invalid when set to a past date |
| 7 | `status` marked invalid when set to an unrecognized value |
| 8 | Save button is disabled while form is invalid |
| 9 | Save button is enabled when form becomes valid |
| 10 | Clicking Save calls `TaskService.create()` exactly once |
| 11 | Loading indicator is visible while saving |
| 12 | Navigates to `/tasks` after successful save |
| 13 | Displays validation error messages from the API (422 response) |
| 14 | Displays a generic error message on unexpected server error (500) |
| 15 | Form values are passed correctly to `TaskService.create()` |
| 16 | Displays a success alert after a successful save |
| 17 | Renders placeholders on all form fields |
| 18 | Renders `status` as a `<select>` (or `mat-select`) |

## Requirements

Use:

- Jasmine + Karma
- Angular `TestBed`
- `HttpTestingController` for service-level HTTP assertions
- Angular Material

Add a comment on each test explaining **why** it exists.

The tests must **not** pass until the production component is implemented in the next prompt.
