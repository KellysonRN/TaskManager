# AI Usage Notes

This repository was scaffolded with a professional, interview-friendly structure that highlights:

- Clean Architecture boundaries
- scalable API and frontend separation
- documentation-first delivery
- containerized local development

## Recommended AI-assisted workflow

1. Use AI to draft use cases, domain entities, and API contracts.
2. Keep business rules in the Application and Domain layers.
3. Prefer explicit interfaces and dependency inversion in Infrastructure.
4. Use documentation under docs/ as the source of truth for architecture decisions and presentation notes.

## Presentation Guidance

When presenting this repository in an interview:

- emphasize separation of concerns
- explain the flow from API to Application to Domain to Infrastructure
- highlight readiness for future growth, testing, and deployment
